# LeafAuto Pro 项目开发文档

**日期**: 2025年4月16日 19:11:05

## 项目概述

### 背景与目标

LeafAuto Pro 是一款基于 PyQt6 构建的桌面应用程序，旨在通过集成自动化任务处理和智能助手功能来提高个人及企业用户的生产力。该应用不仅提供了一系列高级功能，还特别注重用户体验、性能优化和安全性。

#### 主要特点

- **跨平台支持**: 利用PyQt6的优势，LeafAuto Pro能够在Windows、macOS和Linux等多个操作系统上无缝运行。
- **模块化设计**: 应用程序被划分为多个独立但相互协作的模块，便于维护、扩展和升级。
- **高性能与稳定性**: 针对响应速度和稳定性进行了深度优化，确保在高负载情况下依然表现优异。
- **用户友好界面**: 界面设计直观易用，减少用户的学习成本，提高工作效率。

#### 核心功能

- **自动化信息处理**: 自动化日常任务，如数据收集、文件整理、邮件发送等。通过预设规则或用户自定义脚本实现自动化流程。
- **AI 助手**: 提供自然语言处理（NLP）能力，允许用户通过语音或文本与系统进行交互。例如，安排会议、查询天气、设置提醒等。
- **活动管理**: 支持创建、编辑和删除各种活动，具备提醒和日程安排功能。用户可以轻松管理自己的日程表，并接收及时的通知。

#### 性能提升

相较于标准版本，LeafAuto Pro 在以下方面进行了显著改进：
- **更快的响应速度**: 通过优化算法和采用多线程技术，减少了延迟时间。
- **增强的安全性**: 实施了更严格的数据保护措施，包括加密存储和传输中的敏感信息。
- **更多自定义选项**: 用户可以根据自己的需求调整界面布局、快捷键设置等，提供更加个性化的体验。

### 使用场景

无论是个人用户希望提高日常工作效率，还是企业需要自动化其业务流程，LeafAuto Pro 都是一个理想的解决方案。

## 文件结构

LeafAuto Pro 的项目结构采用了模块化设计，以确保易于维护和扩展。以下是主要的目录结构和关键文件描述：
# LeafAuto Pro 文件结构

LeafAuto-Pro/
- **App.py**  
  - 描述: 应用程序的主入口点，负责初始化 `QApplication`、检查单实例运行状态、创建并显示主窗口，并启动主事件循环。
  
- **Application.spec**  
  - 描述: 用于配置 PyInstaller 打包过程的文件，定义了如何将 Python 脚本打包成独立的可执行文件。

- **common.py**  
  - 描述: 包含常用的辅助函数和工具类，例如日志记录、字符串处理等，供其他模块调用。

- **LeafAuto_Pro_version_info.txt**  
  - 描述: 包含应用程序的版本号和其他元数据，用于在打包过程中生成正确的版本信息。

- **LeafProcess.py**  
  - 描述: 负责管理后台进程和处理进程间通信，确保应用程序的稳定性和高效性。

- **split.py**  
  - 描述: 提供大段文字分割功能，支持将大文件分割为多个小文件，并处理这些分割后的文件。

- **SystemInfo.py**  
  - 描述: 获取系统相关信息（如 CPU 使用率、内存占用等），并提供接口以展示这些信息，帮助用户了解当前系统的运行状况。
 
打包命令如下：

```bash
pyinstaller Application.spec
```


## 技术栈

### 1. **Python**
- **版本**: 3.11
- **用途**: Python作为主要编程语言，用于整个应用程序的开发。它提供了丰富的标准库和第三方库支持，使得快速开发复杂的应用程序成为可能。在LeafAuto Pro中，Python被用来编写业务逻辑、数据处理以及与外部服务进行交互。

### 2. **PyQt6**
- **版本**: 6.4.2
- **用途**: PyQt6 是一个功能强大的GUI开发框架，基于Qt 6.x系列。它允许开发者创建现代、跨平台的桌面应用程序。在LeafAuto Pro中，PyQt6用于构建用户界面，提供流畅的用户体验，并确保应用能在不同操作系统上一致地运行。

### 3. **PyInstaller**
- **版本**: 5.6.2
- **用途**: PyInstaller是一个用于将Python应用程序打包成独立可执行文件的工具。它可以处理复杂的依赖关系，并生成适用于Windows、macOS和Linux的可执行文件。在LeafAuto Pro项目中，PyInstaller被用来打包最终的应用程序，使其易于分发和部署。

### 4. **requests**
- **版本**: 2.32.3
- **用途**: requests是一个简单但功能强大的HTTP请求库，广泛用于与外部API进行通信。在LeafAuto Pro中，requests库被用来发送HTTP请求并处理响应，以便与各种在线服务集成。这使得应用程序能够动态获取最新信息和服务更新。

### 5. **pywin32**
- **版本**: 308
- **用途**: pywin32是Windows平台上的扩展库，提供了对Windows API的访问。在LeafAuto Pro中，它主要用于创建和管理互斥锁，确保程序的稳定性和安全性。此外，pywin32还用于实现进程间通信和其他系统级操作。

### 6. **QSharedMemory**
- **用途**: QSharedMemory是Qt提供的一个类，用于实现进程间的共享内存。在LeafAuto Pro中，QSharedMemory用于确保应用程序只有一个实例在运行，从而避免资源冲突。这通过在`App.py`中创建一个`QSharedMemory`对象来实现，尝试附加到已存在的共享内存或创建新的共享内存段。

## 运行原理

### 1. **单实例运行**
- **原理**: 使用 `QSharedMemory` 类来确保应用程序只有一个实例在运行。
- **实现**:
  - 在 `App.py` 中创建一个 `QSharedMemory` 对象。
  - 尝试附加到已存在的共享内存，如果成功则显示警告消息并退出。
  - 尝试创建共享内存，如果失败则显示错误消息并退出。

### 2. **UI 设计**
- **原理**: 使用 Qt Designer 设计 UI，然后通过 `pyuic6` 工具将 `.ui` 文件转换为 Python 代码。
- **实现**:
  - 设计 `MainWindow.ui` 和 `Activities.ui` 文件。
  - 使用 `pyuic6` 将 `.ui` 文件转换为 `Ui_MainWindow.py` 和 `Ui_Activities.py`。

### 3. **多线程处理**
- **原理**: 使用 `QThread` 类来创建和管理多线程，确保 UI 界面的响应性。
- **实现**:
  - 在 `Thread.py` 中定义多个线程类。
  - 在主窗口中启动和管理这些线程，以执行耗时的任务而不阻塞主线程。

### 4. **AI 助手**
- **原理**: 使用外部 AI API 进行对话处理。
- **实现**:
  - 在 `AiAssistant.py` 中调用 AI API。
  - 处理 AI 返回的结果并显示在界面上，提供自然语言处理和智能响应功能。
